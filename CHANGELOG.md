## [Unreleased]

### Fixed
- **WPM readback now interprets FAVEWPM correctly instead of treating EEPROM byte 1 as the current speed.** `parseLoadDefaults()` was reading `eeprom[1]` as the device's current WPM, matching the Load Defaults command's wire order (`data.txt:697`). But the actual WK3 EEPROM layout at `data.txt:2500-2517` stores `FAVEWPM` at byte 1 — the "favorite speed" that serves as the pot's lower bound when set, or `0` for potlock mode where the pot operates over its full range. Favorite speed isn't a separate mode in WK3; it's a floor on the pot range (see the `S` command at `data.txt:1557-1558` — "This speed is activated when the speed pot is turned fully counterclockwise. The speed pot will act normally above this setting"). `parseLoadDefaults()` now reads byte 1 as `FAVEWPM` and seeds `settings.wpm` accordingly: if `FAVEWPM > 0`, use it directly (clamped to `[MIN_WPM, maxWpm]`); if `FAVEWPM == 0` (potlock), fall back to the midpoint of `[MIN_WPM, maxWpm]` as a neutral placeholder. `maxWpm` continues to be computed from bytes 6+7 (`MINWPM` + `WPMRANGE`), which the layout table confirms *are* the same in both the Load Defaults command and the EEPROM map. The first Set Speed (`02 nn`) still only fires when the user moves the slider — no wire writes at connect time. TODO comment added about capturing the `0b10xxxxxx` speed-pot bytes from `runReadLoop` as a future enhancement that would give potlock devices a real live-speed reading instead of the midpoint placeholder.

### Changed
- **WKUSB connect now reads device state from EEPROM instead of pushing hardcoded init bytes.** New init flow in `src/lib/wkusb.js#connect()`: `Set WK3 Mode (00 14)` → `Dump EEPROM (00 0C)` (both admin commands, accepted in standalone mode pre-host-open) → parse the first 15 bytes per the Load Defaults order from `data.txt` → hydrate `settings` from the parsed values → `Host Open (00 02)` → only *then* apply conditional corrections. The Mode register byte is rewritten only if bit 6 (paddle echoback) is 0 or bit 2 (serial echoback) is 1, and the offending byte is logged via `console.warn` before the corrected value is sent. The PinCfg byte is rewritten only if bit 0 (PTT enable) is 1, with the same log-then-correct pattern. Every other field — WPM, max WPM, sidetone Hz, key mode, paddle swap, autospace, contest spacing, paddle watchdog, ultimatic priority, hang time, KeyOut1/2, sidetone enable — is now derived from EEPROM and the device is the source of truth. New helpers: `parseLoadDefaults`, `armEepromCapture`/`finishDumpCapture`, `needsModeCorrection`/`correctModeRegister`, `needsPinCfgCorrection`/`correctPinCfg`, `hex`. New read-loop mode `EEPROM_CAPTURE` collects raw 256-byte dumps with a 3 s timeout (256 bytes ≈ 2.13 s at 1200 baud). On timeout the bring-up tears down the port and surfaces an error to the UI rather than falling back to the old hardcoded init.
- **Settings panel grays out when no device is connected.** Every input in `src/components/WkusbSettings.jsx` (WPM, max WPM, key mode, sidetone toggle, sidetone Hz, sidetone volume) now uses `disabled={!connected}` (or `!connected || !settings.sidetoneEnabled` for the dependent sidetone controls). The disconnected hint message updated from "changes are saved and applied on next connect" to "Connect a device to change settings — values are read live from the keyer's EEPROM."

### Removed
- **localStorage caching of WKUSB settings (except sidetone volume).** The old `morseChat.wkusbSettings` blob and the `loadSettings()` / `saveSettings()` helpers in `src/lib/wkusb.js` are gone. A one-time `migrateLegacySettings()` runs at module load to drop the stale key. Setters (`setWpm`, `setMaxWpm`, `setSidetoneEnabled`, `setSidetoneHz`, `setKeyMode`) early-return when `!connected || !writer` and no longer touch localStorage. **Sidetone volume is the lone exception:** it's not in the EEPROM Load Defaults map and the manual doesn't publish its EEPROM address, so it still persists to `morseChat.wkusbSidetoneVolume`. Default volume changed from 3 → 1 because the device's actual volume is unknown until the user explicitly sets it (and a low default avoids surprising loud sidetone on first use). The hardcoded `INIT_PIN_CFG` / `INIT_MODE_REG` connect-time writes from the previous Unreleased Fixed entry are also gone.

### Added
- **Help toggle in the app header that reveals handwritten lobby hints on demand.** New circular "?" button to the left of the callsign tag in `app-header`. Clicking it flips a `helpOpen` state in `App.jsx`; while open, three short hints rendered in the "Just Another Hand" Google font appear above their respective lobby cards: above WKUSB connection ("First, connect your WK3 device and change any settings you need"), above the online toggle ("Go online when you're ready to chat"), and above the user list ("Other online users will show here"). Help mode also reveals a free-flowing Notes paragraph below the user list — same handwritten font, no card wrapper — with a hardware-compatibility caveat and an external link to the K1EL WKUSB-AF & CPO product page (https://www.k1elsystems.com/WKUSB_AF.html). Hints are hidden by default and disappear again when the button is clicked a second time. Each lobby card is wrapped in a `.hinted-section` flex container so the hint sits tight against its card when shown. New CSS classes `.hint-handwritten` (with link styling for the in-text WKUSB-AF link), `.hinted-section`, and `.help-toggle` (`.active` variant for the toggled-on state) in `src/styles.css`; Google Fonts CSS imported via `@import` at the top of the stylesheet.
- Vercel Analytics: `@vercel/analytics` dependency, `<Analytics />` mounted in `src/main.jsx` alongside `<App />`. Enable in the Vercel project dashboard for the data to start flowing.

### Added
- **Sidetone volume control re-added (corrected wire byte).** Set Sidetone Volume admin command, exposed via a Low/2/3/High dropdown in the WKUSB settings panel — `n = 1 (low)`, `2`, `3`, `4 (high)`. Default is level 3. Plumbing: `setSidetoneVolume` setter, `sidetoneVolume` field in `defaultSettings` / `getSettings`, hook export, App wiring, dropdown in `WkusbSettings`. **Wire bytes are `00 19 nn`, NOT `00 24 nn`.** The K1EL `wkUSBreadME.md` lists this as `25: Set Sidetone Volume <00><24><n>`, but the `<24>` is a typo — the K1EL convention is "decimal label number = hex command byte" (e.g. "20: Set WK3 Mode" → `0x14` because 20 dec = 0x14 hex), so the correct byte is `0x19` (= 25 decimal). Two prior attempts using the typo'd `0x24` errored the device, which is why the long-standing memory said this command was hardware-broken. It wasn't — we were sending a different admin command entirely. Audit confirmed every other wire command in the codebase already follows the correct convention.

### Removed
- **PTT enable toggle.** The "PTT enable (transmit on radio)" checkbox in the WKUSB settings panel and its full plumbing (`setPttEnabled` setter, `pttEnabled` field in `defaultSettings` / `getSettings`, `onPttEnabledChange` prop on `WkusbSettings`, hook export, App wiring, `computePinConfig` bit 0 handling). PinCfg bit 0 is now hardcoded to 0 in `computePinConfig()` and is already 0 in `INIT_PIN_CFG` (`0x02`), so the device-side behavior is unchanged from the just-shipped init. Comment in `ChatView` that referenced PTT-driven transmit removed.

### Changed
- **Sidetone frequency UI now spans the full WK3 range (500–4000 Hz).** Previously capped at 1500 Hz, which was an arbitrary practical limit. WK3 mode is continuously adjustable across the full band per the WinKeyer 3 datasheet, and there's no hardware reason to constrain the slider. Formula (`62500/freq_hz`) and Set Sidetone Control (`01 nn`) wire command unchanged. The `WkusbSettings` slider picks up the new bound automatically via `settings.maxSidetoneHz`.

### Fixed
- **Paddle keying did nothing on one side of a two-way test.** Root cause: `connect()` was only sending Admin Host Open and trusting the device's standalone EEPROM Mode register, but Mode bit 6 (paddle echoback) was off in one operator's hardware — so paddle bytes never echoed back to the host, the read loop never saw them, and `onChar` never fired. Keyboard typing still worked because it bypasses the read path. Fix: `connect()` now pushes a fixed init sequence after Host Open — Admin Set WK3 Mode (`00 14`), PinCfg `09 02` (sidetone on, all key outputs and PTT off), and Mode register `0E 50` (paddle echo on, iambic A, serial echo off, all else off). The bytes are intentionally hardcoded (not derived from `settings`) so the routing-critical bits are guaranteed regardless of stale localStorage. Reverses the prior "minimal connect" policy from the previous Unreleased Removed entry. `defaultSettings.keyOut2Enabled` flipped from `true` → `false` so `computePinConfig()` defaults match the new init byte.

### Changed
- Lobby user list is now visible regardless of the user's own online state. The "Appear online" toggle still controls whether *you* show up to others, but you can browse who's online and initiate connection requests without first making yourself visible. `OnlineToggle` copy updated to reflect this.

### Fixed
- **`Set PinCfg` and `Set WinKeyer Mode` no longer clobber unrelated bits.** The K1EL protocol writes the entire byte for both registers and offers no register read-back, so the previous live setters were silently zeroing hang time, ultimatic priority, paddle swap, autospace, contest spacing, and the paddle watchdog disable bit on every toggle. The settings object now tracks the **full** state of both registers; `computeModeRegister()` and `computePinConfig()` build the byte from every field. Defaults mirror the working reference at `newqrzframe/public/wkusb-manager.js` (`defaultModeRegister = 0x50`, `defaultPinConfig = 0x06`); a default-state round-trip produces exactly those bytes. Bits 6 (paddle echoback ON) and 2 (serial echoback OFF) of the mode register remain forced because they're load-bearing for the chat→pair routing model.

### Added
- WKUSB settings panel (collapsible, below the connection bar) with sliders/toggles for WPM speed, **max speed (slider upper bound)**, **key mode (Iambic A/B/Ultimatic/Bug)**, sidetone on/off, sidetone frequency (500–1500 Hz), and PTT enable. Settings persist in `localStorage` (`morseChat.wkusbSettings`) and are applied live to the device when connected — or staged for next connect when offline.
- Public setter API on `src/lib/wkusb.js`: `setWpm`, `setMaxWpm`, `setSidetoneEnabled`, `setSidetoneHz`, `setKeyMode`, `setPttEnabled`, plus `getSettings` / `onSettings` for state subscription. Wire commands: WPM = Set Speed (`02 nn`), max WPM = Setup Speed Pot (`05 min range pad`), sidetone freq = Sidetone Control (`01 nn`, byte = 62500 / Hz), key mode = Set WinKeyer Mode (`0E nn`) with the mode-register field rebuilt from `MODE_REGISTER_BASE | KEY_MODE_BITS[mode]`, sidetone enable + PTT enable rebuild PinCfg from `PIN_CONFIG_BASE | flags` and write Set PinCfg (`09 nn`).

### Removed
- **Sidetone volume control.** The Admin Set Volume command (`00 24 nn`) was breaking connection on the user's hardware. Setting, setter, hook export, panel UI, and the post-Load-Defaults volume write are all gone.
- **All init defaults pushed at connect time.** The `connect()` bring-up no longer sends `Set WK3 Mode` (`00 14`) nor the `Load Defaults` block (`0F` + 15 values). After the host-open handshake the device retains whatever settings it had standalone. User-set values are only pushed when the user actively changes a control in the settings panel — that path uses the focused per-setting commands which are known-safe.

### Notes
- Setting max WPM clamps the current WPM downward if it exceeded the new max, then pushes both Setup Speed Pot and Set Speed in one go so the device and slider stay in sync.

### Changed
- Branded the app as "somber's morse chat" (browser tab title and in-app header).
- `src/lib/wkusb.js` is no longer a stub: real Web Serial integration for WKUSB / WKMini / WKUSB-AF using the K1EL WinKeyer 3 protocol. Connect at 1200 8N1, host-open handshake (`00 02`) with revision-code discard, then `00 14` (Set WK3 Mode) followed by Load Defaults (`0F` + 15 values). Mode register `0x50` enables paddle echoback and disables serial echoback so we never receive our own writes back. Read loop classifies bytes by tag (status / speed-pot / echo) and dispatches paddle-sourced ASCII to `onChar` listeners. Adds `onState`, `clearBuffer`, and `isBusy` to the public API.
- ChatView now plays remote `morse-text` on the local WKUSB (so the operator hears their partner's keying), and locally typed/pasted characters also go to the WKUSB so they're heard/transmitted alongside being published. Paddle-sourced characters are sent to the pair channel but never looped back to the device.
